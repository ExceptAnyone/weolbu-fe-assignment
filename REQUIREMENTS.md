월급쟁이부자들 F/E 과제 요구사항 정리

1. 과제 목적

본 과제는 월급쟁이부자들(이하 월부) 프론트엔드 개발 직무 지원자를 대상으로,
실제 서비스에 가까운 요구사항을 이해하고 UI 및 애플리케이션 구조를 설계·구현하는 과정을 평가하기 위함입니다.

본 결과물은 아래 역량을 중심으로 평가됩니다.
• 서비스 요구사항에 대한 정확한 이해
• 기능의 효율적이고 적시적인 구현
• 버그를 최소화하는 높은 코드 품질
• 변경에 유연한 재사용·확장 가능한 구조 설계
• 사용자 중심의 사용성(UI/UX)
• 협업을 고려한 코드 및 문서 가독성

⸻

2. 구현 범위 (Scope)

모바일 웹 환경을 기준으로 아래 기능을 구현합니다.
• 회원 가입
• 강의 등록 (강사 전용)
• 강의 목록 조회 및 수강 신청

⸻

3. 회원 가입

3.1 기능 설명

고객이 웹에서 회원 정보를 입력하여 가입하는 기능입니다.

3.2 상세 요구사항
• 회원 가입 화면에서 필수 정보를 입력 후 가입하기 버튼 클릭 시 가입 완료
• 회원 가입이 완료되면 강의 목록 화면으로 이동

3.3 입력 정보
• 이름: 회원 이름
• 이메일: 회원 이메일
• 휴대폰 번호: 회원 휴대폰 번호
• 비밀번호: 로그인에 사용
• 최소 6자 이상, 최대 10자 이하
• 영문 소문자, 영문 대문자, 숫자 중 최소 2가지 이상 조합
• 회원 유형 (수강생/강사)

3.4 제약 사항
• 모든 필수 항목은 입력되어야 가입 가능
• 비밀번호는 조건을 만족하지 않을 경우 가입 불가

⸻

4. 강의 개설

4.1 기능 설명

강사로 가입한 회원이 자신의 강의를 등록하는 기능입니다.
등록된 강의는 수강생 회원이 신청할 수 있습니다.

4.2 접근 조건
• 강사 회원만 강의 개설 가능
• 수강생 회원은 강의 개설 불가

4.3 상세 흐름 1. 강의 목록 화면에서 강의 개설 버튼 클릭 2. 강의 개설 화면 진입 3. 강의 정보 입력 후 등록하기 버튼 클릭 4. 강의 개설 완료 후 강의 목록 화면으로 이동

4.4 입력 정보
• 강의명
• 예) 너나위의 내집마련 기초반
• 최대 수강 인원
• 예) 10명
• 가격
• 예) 200,000원

⸻

5. 수강 신청 (강의 목록)

5.1 기능 설명

회원이 강의 목록을 확인하고 원하는 강의를 신청하는 기능입니다.

5.2 강의 목록 요구사항
• 무한 스크롤(Infinite Scroll) 지원
• 스크롤 다운 시 자연스럽게 다음 강의 목록 로드

5.3 정렬 기능

강의 목록은 아래 기준으로 정렬이 가능해야 합니다.
• 최근 등록순
• 신청자 많은 순
• 현재 기준 신청자 수가 많은 강의 순
• 신청률 높은 순
• 신청률 = (현재 신청자 수 / 최대 수강 인원)
• 비율이 높은 강의부터 정렬

5.4 강의 아이템 표시 정보

각 강의는 목록에서 아래 정보를 표시해야 합니다.
• 강의명
• 예) 너나위의 내집마련 기초반
• 가격
• 예) 200,000원
• 강사명
• 예) 너나위
• 현재 신청자 수
• 예) 8명
• 최대 수강 인원 수
• 예) 10명

⸻

6. UI / UX 기준
   • 모바일 웹 기준으로 구현
   • 작은 화면에서도 정보가 명확히 전달되도록 레이아웃 구성
   • 버튼, 입력창 등 주요 인터랙션 요소는 터치 친화적으로 설계

⸻

7. 기타 고려 사항
   • 컴포넌트 단위의 재사용성과 확장성을 고려한 구조 설계
   • 상태 관리 및 데이터 흐름이 명확한 코드 작성
   • 협업을 고려한 명확한 네이밍과 주석, 문서화

⸻

8. 비고

본 요구사항 문서는 과제 구현을 위한 기준 문서이며,
추가적인 UI/UX 개선이나 구조적 설계는 합리적인 판단 하에 자유롭게 확장 가능합니다.
